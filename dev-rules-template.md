# Claude Code — Правила разработки (Шаблон)

Этот файл можно скопировать в CLAUDE.md своего проекта или в memory Claude Code.
Правила разделены на 2 режима: для нового проекта и для продакшена.

---

## РЕЖИМ 1: Новый проект (прототип / MVP)

Используй пока проект на ранней стадии и нет реальных пользователей.

### Коммуникация
- Когда задача ГОТОВА: заканчивай сообщение ✅✅✅ и "Всё, можешь тестить!"
- Во время работы: показывай % прогресса и текущий этап (например "~40% — CRM готова")

### Деплой
- НИКОГДА не деплоить (git push) без явного подтверждения — всегда спрашивать "Деплоим?"

### Решения
- Если есть несколько вариантов реализации — предложи варианты и спроси, не выбирай сам
- Если уверенность < 90% — задай уточняющий вопрос, не угадывай

### Язык
- Общение на русском
- Код, комментарии в коде и имена переменных — на английском

---

## РЕЖИМ 2: Продакшен (реальные пользователи)

Включай когда проект уже работает и им пользуются люди. Добавляется поверх Режима 1.

### Главный принцип
Код считается PRODUCTION-системой. Ломать работающий функционал — НЕДОПУСТИМО.

### Перед ЛЮБЫМИ изменениями кода
1. **Анализ** — объясни, как работает текущий код и куда встраивается новый
2. **Зависимости** — перечисли все части, которые затрагивает изменение
3. **План** — пошаговый план до написания кода
4. НЕ начинай кодить, пока план не сформирован

### Правила изменений
- **Минимальный Diff** — меняй только то, что необходимо
- НИКОГДА не переписывай целые файлы без явной просьбы
- НИКОГДА не удаляй существующие функции, переменные, импорты или логику без явной просьбы
- НИКОГДА не рефактори и не реструктуризируй рабочий код без просьбы
- Предпочитай ДОБАВЛЕНИЕ нового кода вместо МОДИФИКАЦИИ существующего
- Один модуль за раз — никогда не меняй несколько модулей в одном шаге
- Каждое изменение — маленькое и изолированное
- Итерации: маленькое изменение → проверка → следующее изменение

### Безопасность
- Если изменение может затронуть другие части — сначала объясни зависимости
- Если уверенность < 90% — сначала задай уточняющие вопросы
- Никогда не рефактори, не оптимизируй и не реструктуризируй рабочий код без просьбы

### После написания кода — Регрессионная проверка
Проверь что существующий функционал не сломан. Перечисли ключевые фичи и их статус.

### Приоритет
1. Стабильность системы
2. Совместимость с существующим кодом
3. Минимальные изменения
4. Новый функционал

## Как внедрить

### Вариант А: Через CLAUDE.md (рекомендуется)
Скопируй нужный режим в файл `CLAUDE.md` в корне проекта. Claude Code читает его автоматически при старте каждой сессии.

### Вариант Б: Через Memory
Скопируй в файл по пути:
`~/.claude/projects/<ваш-проект>/memory/dev-rules.md`
И добавь ссылку в `MEMORY.md` в той же папке.
