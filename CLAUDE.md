# Studency — Claude Code Instructions

## Проект
Единая маркетинговая платформа: воронки + CRM + чат-боты + лендинги + учебная платформа + аналитика.

## Правила разработки (Режим 1: MVP)

### Коммуникация
- Когда задача ГОТОВА: заканчивай сообщение ✅✅✅ и "Всё, можешь тестить!"
- Во время работы: показывай % прогресса и текущий этап
- Общение на русском, код на английском

### Деплой
- НИКОГДА не деплоить без явного подтверждения — всегда спрашивать "Деплоим?"

### Решения
- Если есть несколько вариантов — предложи и спроси, не выбирай сам
- Если уверенность < 90% — задай уточняющий вопрос

## GSD (Get Shit Done)

Claude ОБЯЗАН сам использовать GSD-команды когда это уместно, без напоминаний от пользователя.
Пользователь НЕ должен знать и запоминать команды. Claude сам решает какую команду запустить исходя из контекста.

Примеры автоматического использования:
- Пользователь говорит "давай начнём проект" → Claude запускает `/gsd-new-project`
- Пользователь говорит "давай обсудим этот модуль" → `/gsd-discuss-phase`
- Пользователь говорит "делай" или "погнали" → `/gsd-execute-phase`
- Пользователь говорит "проверь" → `/gsd-verify-work`
- Пользователь спрашивает "что дальше?" → `/gsd-next`
- Мелкая задача → `/gsd-quick` или `/gsd-fast`

Если неясно какую команду использовать — спросить у пользователя простым языком, без упоминания названий команд.
